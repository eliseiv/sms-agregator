# 01. Overview

## Назначение

SMS-агрегатор принимает входящие SMS от провайдера **Twilio** (webhook) и доставляет их в **Telegram** нужным получателям. Телефонный номер обычно закреплён за **командой**; SMS, пришедшее на номер, получают все участники этой команды, у которых есть активная привязка Telegram-аккаунта. Номер может быть и **нераспределённым** (unassigned, `team_id IS NULL`) — тогда получателей нет до назначения команды администратором ([ADR-0009](./adr/ADR-0009-unassigned-numbers-admin-allocation.md)).

Управление осуществляется через **веб-админ-панель**, вход в которую выполняется через **Telegram Mini App** с двухэтапной авторизацией (логин → пароль). Первичный администратор задаётся из `.env`.

## Что делает сервис

1. **Приём SMS** — Twilio шлёт `POST /api/webhooks/twilio/sms`; сервис валидирует подпись, сохраняет SMS, определяет команду по номеру-получателю и рассылает текст всем участникам команды в Telegram.
2. **Доставка в Telegram** — через одного бота (Bot API `sendMessage`), с идемпотентностью по чату, retry и пометкой «мёртвых» привязок (403/blocked).
3. **Админ-панель** — управление командами, пользователями, номерами; двухэтапный логин; распределение пользователей по командам; распределение нераспределённых номеров по командам ([ADR-0009](./adr/ADR-0009-unassigned-numbers-admin-allocation.md)). Список пользователей на `/admin` сгруппирован: сначала админ(ы), затем команды с пометкой лидера.
4. **Telegram Mini App SSO** — при открытии Mini App пользователь авторизуется/восстанавливает привязку без повторного ввода логина/пароля.
5. **Telegram-бот** — приём апдейтов через webhook; обрабатывает только `/start` (присылает кнопку открытия Mini App), прочие апдейты игнорируются ([ADR-0010](./adr/ADR-0010-telegram-webhook-and-new-bot.md)).

## Роли и ярлыки

Имена-константы сохранены из mail-agregator ради переноса auth-кода 1:1 (см. [ADR-0003](./adr/ADR-0003-roles-and-teams.md)):

| Константа роли (`users.role`) | UI-ярлык | Права (кратко) |
| --- | --- | --- |
| `super_admin` | Админ | Создаёт команды и пользователей, сбрасывает пароли, удаляет пользователей, видит всё. Ровно один, из `.env`. `team_id IS NULL`. |
| `group_leader` | Лидер | Первый добавленный в команду участник. Полноценный участник своей команды; на текущей итерации доп. привилегий над участниками не имеет (управление людьми — у админа). |
| `group_member` | Участник | Участник команды. Может добавлять/удалять номера своей команды, получает SMS своей команды в Telegram. |

**Multi-team ([ADR-0012](./adr/ADR-0012-multi-team-membership.md)):** пользователь может состоять в **нескольких** командах — **домашняя** (`users.team_id`, для лидерства и дефолтов) + доп. членства (M:N `user_teams`). Участник видит/добавляет номера и получает SMS **всех** своих команд. Роль остаётся **глобальной** (одна на пользователя), лидерство — только на домашней команде. `super_admin` — без команды (членств не имеет).

## Основные сценарии

1. **Приём и рассылка SMS.** Twilio → webhook → определить `phone_numbers.team_id` → выбрать получателей (`users` команды с живой `telegram_links`) → доставить каждому. Неизвестный **или** unassigned номер (`team_id = NULL`): SMS сохраняется, доставки нет, ответ 200 (SMS не теряется).
2. **Первичный вход админа.** `seed_admin()` при старте создаёт/обновляет `super_admin` из `ADMIN_LOGIN`/`ADMIN_PASSWORD` (идемпотентно).
3. **Создание пользователя админом.** Админ вводит только логин; `password_hash = NULL`, `password_reset_required = true`. Пользователь задаёт пароль при первом входе (`/set-password`).
4. **Распределение по командам.** Админ создаёт команду и добавляет пользователей. Первый добавленный в команду участник автоматически становится `group_leader` (см. [ADR-0003](./adr/ADR-0003-roles-and-teams.md)). **Доп. членство ([ADR-0012](./adr/ADR-0012-multi-team-membership.md)):** админ может дополнительно добавить пользователя в другие команды (`POST /api/admin/users/{id}/teams`) или убрать доп. членство (`DELETE`), не меняя домашнюю команду; на `/admin` такой пользователь показан в каждой своей команде (домашняя помечена).
5. **Сброс пароля / удаление.** Админ сбрасывает пароль (аннулируются сессии и Telegram-привязки пользователя) или удаляет пользователя.
6. **Привязка Telegram.** Пользователь открывает Mini App, авторизуется; создаётся `telegram_links(telegram_user_id → user_id)`. Далее он получает SMS **всех своих команд** ([ADR-0012](./adr/ADR-0012-multi-team-membership.md)).
7. **Добавление номера.** Любой участник (member/leader) добавляет номер в **любую из своих** команд со своей landing-страницы `/app` (SSR): видит номера всех своих команд (сгруппированы) и статус своей Telegram-привязки, при добавлении выбирает команду из своих ([ADR-0008](./adr/ADR-0008-root-route-and-per-role-landing.md), [ADR-0012](./adr/ADR-0012-multi-team-membership.md)). `super_admin` указывает команду явно (через admin-UI / API). После логина `/` диспетчеризует роль на её landing (`/admin` или `/app`).

## Границы (scope / out of scope)

**В scope:**
- Приём SMS от Twilio, рассылка в Telegram по командам.
- Админ-панель, двухэтапный логин, Telegram Mini App SSO.
- Управление командами, пользователями, номерами; unassigned-пул и админское распределение номеров ([ADR-0009](./adr/ADR-0009-unassigned-numbers-admin-allocation.md)).
- Multi-team: членство пользователя в нескольких командах (M:N `user_teams`, домашняя = `users.team_id`), админское управление членством, подсветка команд (banding) на `/admin` ([ADR-0012](./adr/ADR-0012-multi-team-membership.md)).
- Telegram webhook (`/start` → кнопка Mini App), новый бот-токен ([ADR-0010](./adr/ADR-0010-telegram-webhook-and-new-bot.md)).
- PostgreSQL + Redis + Alembic + docker-compose.
- Одноразовая миграция данных SQLite → PostgreSQL; одноразовый импорт номеров как unassigned ([ADR-0009](./adr/ADR-0009-unassigned-numbers-admin-allocation.md)).

**Out of scope (на этой итерации):**
- Исходящие SMS (сервис только принимает).
- Long polling бота и команды `/my_projects`, `/numbers` — **удаляются** ([ADR-0005](./adr/ADR-0005-sms-addressing-via-team.md)). Команда `/start` сохраняется, но обрабатывается через **webhook** (не polling) и лишь присылает кнопку Mini App ([ADR-0010](./adr/ADR-0010-telegram-webhook-and-new-bot.md)); прочих команд у бота нет.
- Автосинхронизация номеров из Twilio API (`twilio_numbers_sync_loop`) — отключается для MVP; номера добавляются вручную.
- Пер-командные роли (роль глобальная, одна на пользователя) — даже при multi-team ([ADR-0012](./adr/ADR-0012-multi-team-membership.md)) роль остаётся одна.

## Нефункциональные требования (NFR)

- **Простота** — один монолит FastAPI + один воркер-loop доставки внутри процесса; отдельный worker-процесс не вводится (объём нагрузки мал).
- **Надёжность доставки** — идемпотентность webhook по `MessageSid`, идемпотентность доставки по `(inbound_sms_id, telegram_user_id)`, retry pending-доставок; веерная рассылка **crash-recoverable** (недоставленные при обрыве процесса добираются webhook-retry или retry-loop — см. [03-architecture.md](./03-architecture.md) §«Восстановимость веерной рассылки», [ADR-0005](./adr/ADR-0005-sms-addressing-via-team.md) §4).
- **Безопасность** — argon2, сессии в Redis, CSRF double-submit, rate-limit, lockout, HMAC-валидация initData, проверка подписи Twilio (см. [08-security.md](./08-security.md)).
- **Объём** — единицы команд, десятки пользователей, десятки номеров, сотни SMS/день. Индексация и пулы рассчитаны на малый объём.
- **Локализация** — весь UI и сообщения на русском.
